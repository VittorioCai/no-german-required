# Security Policy

## Supported versions

Security fixes are applied to the current `main` branch. This project has not published
a stable release yet.

## Reporting a vulnerability

Please do not open a public issue for vulnerabilities or suspected secret exposure.
Use GitHub's private vulnerability reporting feature on the repository Security tab.
Include affected files, reproduction steps, impact, and any suggested mitigation.

Never include real LLM keys, SMTP app passwords, Telegram tokens, email addresses, or
private application data in a report. Rotate any credential immediately if you believe
it has been exposed.
